# InnerLife v2：自主经历层

## 1. 问题

v1 已经可以完成跨会话内部连续性，但输入主要来自：

- 用户对话
- Memoria 中的事实
- Continuity 中的共同位置

这意味着 Agent 虽然能在用户离开后继续消化，原材料仍然主要由用户提供。

v2 要解决：

> Agent 是否能拥有一段没有用户参与、但真实发生且能够改变自己的经历？

## 2. 核心定义

自主经历不是模型生成一句“我今天看了什么”。

一次有效自主经历必须同时满足：

1. 选择来自 Agent 自己已有的兴趣、未完成问题或长期关注。
2. 接触了真实存在的外部材料。
3. 保存了来源地址、获取时间和实际读取内容的证据。
4. Agent 可以选择不读取、不形成变化、不分享。
5. 经历先被记录，再由 InnerLife 决定是否产生内部变化。

## 3. 系统位置

```text
外部真实世界
  └─ 文章、论文、项目动态、公开资料
          ↓
自主经历层
  ├─ 获取候选
  ├─ 根据自身关注选择或跳过
  ├─ 实际读取
  └─ 保存经历证据
          ↓
InnerLife
  ├─ 消化经历
  ├─ 更新兴趣或问题
  └─ 形成或不形成待分享念头
```

自主经历层负责“她接触了什么”。

InnerLife 负责“这件事是否改变了她”。

## 4. 输入来源

第一版支持显式允许的公开来源：

- RSS / Atom 订阅
- 公开文章页面
- 公开论文摘要
- 公开项目动态

来源配置属于 Agent profile。每个 agent 可以订阅不同来源。

默认不做：

- 不随机抓取全网新闻
- 不访问登录后内容
- 不读取私人账号、邮件或聊天
- 不自动执行现实世界写操作
- 不把搜索结果标题当成已经阅读

## 5. 自主选择

候选材料出现后，Agent 根据以下内容决定：

- 当前兴趣
- 未完成问题
- 最近已经看过什么
- 是否有实质新意
- 是否值得占用一次注意力

允许返回：

```json
{
  "selected": false,
  "reason": "今天没有值得继续看的材料"
}
```

不允许为了维持活跃而强制选择。

## 6. 真实阅读

选中后，系统必须实际获取页面。

保存：

- 来源名称
- 标题
- URL
- 发布时间（来源提供时）
- 获取时间
- 内容摘要
- 内容指纹
- 实际读取的文本片段

获取失败时记录失败，不生成经历。

## 7. 经历形成

阅读完成后，模型只基于实际文本生成：

```json
{
  "experience_summary": "实际接触了什么",
  "why_it_mattered": "为什么和当前关注有关",
  "new_questions": [],
  "possible_revision": "",
  "worth_digesting": true
}
```

如果材料没有带来东西：

```json
{
  "worth_digesting": false,
  "reason": "内容与已知理解重复"
}
```

只有 `worth_digesting=true` 才进入 InnerLife 输入队列。

## 8. 与用户的关系

自主经历可以完全与用户无关。

用户不是经历的触发者，也不自动成为分享对象。

即使形成变化，也默认只保存在内部。是否分享仍由 InnerLife 的分享规则决定。

## 9. 调度

后台根据以下条件决定是否探索：

- Agent 已启用自主经历
- 距离上次探索超过最小间隔
- 当前没有活跃对话
- 没有处于失败退避期
- 当日探索次数没有超过限制

建议初版：

- 每个 Agent 每天最多 2 次探索
- 每次最多选择 1 份材料
- 没有合适材料时不调用阅读和消化

## 10. 数据

新增三类数据：

### source_subscriptions

Agent 被允许访问的公开来源。

### exploration_runs

每次自主探索尝试，包括没有选择、获取失败和成功。

### autonomous_experiences

真实形成的自主经历及其证据。

## 11. 完成标准

v2 第一版完成必须证明：

1. 没有用户输入时，Agent 可以自主发起一次探索。
2. 选择理由来自 Agent 自己的兴趣或未完成问题。
3. 实际读取了公开外部材料。
4. 经历保存真实来源和内容证据。
5. 相同材料不会重复形成经历。
6. 无合适材料时可以保持沉默。
7. 获取失败不会生成虚构经历。
8. 不同 agent 可以选择不同材料。
9. 自主经历可以进入 InnerLife 并影响下一次会话简报。
10. 用户没有参与的经历不会被描述成共同经历。

## 12. v2 第一版非目标

- 不做自由浏览器 Agent
- 不做社交账号
- 不主动发帖、评论或发送消息
- 不与其他 Agent 自动聊天
- 不自动修改代码项目
- 不追求每天一定发生经历
- 不把外部信息流做成推荐系统

第一版只证明：

> 她能够根据自己的关注，真实地选择、接触和消化一件与你无关的外部事物。
