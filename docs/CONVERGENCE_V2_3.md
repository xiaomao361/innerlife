# InnerLife v2.3：内在状态收敛层

## 1. 问题

长期运行后，以下内容会持续增加：

- 内部变化
- 自主经历
- 未完成问题
- 会话、消化、探索和分享记录

历史增长本身不是错误。真正的问题是日常上下文如果同步增长，Agent 会逐渐被旧内容淹没。

## 2. 目标

v2.3 不直接删除历史，而是把数据分成三层：

- 活跃层：最近、重要、仍会影响当前判断。
- 摘要层：多条旧内容收敛成稳定认识。
- 归档层：退出日常上下文，但仍可追溯。

核心约束：

> 数据可以持续增长，但 Agent 每天需要读取和处理的活跃内在状态必须保持有限。

## 3. 收敛对象

### 内部变化

相似或已被后续理解覆盖的旧变化，可以归档并形成一条稳定摘要。

### 自主经历

已经消化完成、主题重复或不再直接影响当前关注的经历，可以归档。来源和证据仍保留。

### 未完成问题

- `open`：当前仍在推进。
- `dormant`：暂时降温，不进入日常 briefing。
- `resolved`：已经解决，从活跃状态退出。

### 操作记录

会话、消化、探索、分享和收敛记录继续作为审计历史保存，但不会全部进入模型上下文。

## 4. 收敛触发

后台只在以下条件满足时运行：

- 没有活跃会话。
- 活跃内部变化、经历或问题超过 profile 阈值。
- 距离上次收敛超过最小间隔。
- 当前没有待处理输入。

也可以手动运行，用于检查和维护。

## 5. 模型权限

模型只能从系统提供的“可归档候选”中选择，不能归档最新保留区内容。

一次收敛可以：

- 生成一条稳定摘要。
- 归档若干内部变化。
- 归档若干自主经历。
- 将若干问题降温。
- 将有明确依据的问题标记为已解决。
- 判断暂时不需要变化。

所有摘要必须引用真实来源。

## 6. 日常上下文

briefing 默认只返回：

- 有上限的活跃内部变化。
- 有上限的活跃自主经历。
- 状态为 `open` 的未完成问题。
- 最近的稳定摘要。

归档内容仍可通过历史接口查询。

自主探索选择材料时，也会参考稳定摘要，避免把已经形成的长期认识反复当作新发现。

## 7. 默认阈值

- 活跃内部变化最多 40 条。
- 活跃自主经历最多 20 条。
- 活跃未完成问题最多 12 条。
- briefing 返回最近 8 条内部变化、5 条经历和5条摘要。
- 收敛最短间隔 24 小时。
- 每次最多归档 20 条内部变化和 10 条经历。

每个 Agent 可在 profile 中独立调整。

## 8. 完成标准

1. 超过阈值后后台可自动触发收敛。
2. 最新保留区内容不能被归档。
3. 收敛摘要有真实来源。
4. 归档内容退出 briefing。
5. 归档内容仍可查询。
6. dormant 问题退出日常 briefing。
7. 收敛失败不改变原状态。
8. 收敛过程有完整记录。
9. 旧数据库自动升级。
10. 活跃上下文规模保持在 profile 阈值附近。
