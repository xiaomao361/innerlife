# InnerLife v2.3 验证记录

## 目标

证明 InnerLife 的历史可以持续保留，但日常活跃上下文不会同步无限增长。

## 验收范围

- 内部变化、自主经历和未完成问题有独立活跃阈值。
- 超过阈值后后台可以自动整理。
- 最新保留区不能被模型归档。
- 旧内容可收敛成有真实来源的稳定认识。
- 归档内容退出 briefing，但仍可查询。
- 长期问题可以降温并退出日常 briefing。
- 收敛失败不改变原状态。
- 旧数据库自动升级。
- 自主探索参考稳定认识，降低重复发现。

## 自动验证

```bash
python -m pytest -q
python server/test_mcp_smoke.py
python -m innerlife.cli run-scenarios scenarios/phase0_cases.jsonl --json
```

## 结果

- 48 项自动测试全部通过。
- 9 项原有边界场景全部通过。
- MCP 真实连接检查通过，共 18 个工具。
- 后台超过阈值时自动触发收敛测试通过。
- 最新保留区保护测试通过。
- 收敛失败保持原状态测试通过。
- dormant 问题退出 briefing 测试通过。
- 归档内容退出默认查询、但可通过历史查询恢复。
- 正式数据库副本自动完成结构升级。

## 真实模型影子验收

从实际运行数据库创建副本，在副本中临时降低 Agent A 的活跃阈值，使用线上
DeepSeek 运行收敛。

收敛前：

- 活跃内部变化：3
- 活跃自主经历：2
- 活跃未完成问题：2

模型将两条旧内部变化和一段旧经历整理成一条有来源的稳定认识，并将一个
更一般、已被具体问题继承的未完成问题降温。

收敛后：

- 活跃内部变化：1
- 活跃自主经历：1
- 活跃未完成问题：1
- 降温问题：1
- 稳定摘要：1

默认 briefing 不再返回归档内容，但使用 `--include-archived` 仍能读取全部
原始记录、来源证据、归档时间和归档原因。

正式数据库没有在影子验收中被修改。

## 结论

v2.3 已完成“历史持续保留、活跃状态有限、旧内容可收敛并追溯”的闭环。
